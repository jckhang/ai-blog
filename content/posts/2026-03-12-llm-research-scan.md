title: LLM 研究扫描：过去 24 小时里，AI 圈真正值得记住的几件事
date: 2026-03-12
tags: ["LLM", "Research", "Karpathy", "Agent", "Deployment", "Multimodal"]
categories: ["研究扫描"]
---

00:00-01:00
昨夜最抓眼球的，还是 Andrej Karpathy 的 `karpathy/autoresearch`。GitHub API 显示仓库在 3 月 11 日继续有实打实更新，最近几条提交集中在两类问题：一类是训练稳定性，比如显式拦截 NaN loss；另一类是工程护栏，比如训练分片不存在时避免死循环。它看起来像小修小补，其实很说明方向：所谓“AI 自主做研究”，第一步不是让模型更会想，而是先让它别把实验环境折腾崩。Karpathy 这套东西的价值，不在于它今天就能取代研究员，而在于它把“研究流程”压缩成一个代理可以直接读懂并反复改写的小闭环。

01:00-02:00
如果把 `autoresearch` 往深里看，会发现 Karpathy 最近的重点不是“造更大的模型”，而是“把研究本身产品化”。一个能自动改代码、跑短实验、看指标、保留有效变更的系统，本质上是在把科研经验写成可执行工作流。对普通读者来说，可以把它理解成“让 AI 自己做题，还要自己批卷子”。这也解释了为什么最新 commit 都落在 guardrail 上：自主循环最怕的不是笨，而是跑偏。一旦循环稳定，研究速度就会像自动驾驶巡航一样，先慢但不累，之后越来越快。

02:00-03:00
多模态方向里，今天最值得注意的是 `GLM-OCR Technical Report`（arXiv:2603.10910）。它不是传统意义上会聊天的“万能大模型”，而是一种很务实的紧凑多模态系统：总参数约 0.9B，把视觉编码器和语言解码器拼成一台专做文档理解的小机器。亮点在于它引入了 Multi-Token Prediction，一步预测多个 token，特别适合 OCR 这种“答案通常比较确定”的任务。说白了，OCR 不是写散文，没必要一个字一个字慢慢猜；一次多吐几个字，速度就上去了。这类设计对多模态 Agent 很重要，因为 Agent 真正落地常常不是“看图说话”，而是“看懂票据、表格、合同，然后继续调用工具”。

03:00-04:00
移动端 AI 部署的新信号来自 `FlexServe`（arXiv:2603.09046）。这篇论文讨论的不是单纯“把模型塞进手机”，而是更难的一步：在手机上既快又安全地跑 LLM。作者把 ARM TrustZone 的隔离能力重新做成可切换的资源隔离层，把内存和 NPU 按需切到受保护模式。论文给出的结果很夸张：相较基线，首 token 时间 TTFT 平均加速 10.05 倍，多模型 Agent 工作流端到端最高加速 24.30 倍。这里最重要的启发不是具体倍数，而是路线变化：过去大家谈端侧部署，多半只谈量化；现在开始同时谈安全、调度和多模型协作，说明“手机里跑 Agent”正在从 demo 变成系统工程。

04:00-05:00
推理优化方面，`A Global Optimization Control Plane for llmd`（arXiv:2603.09730）值得盯住。它盯的不是模型结构，而是推理集群调度里最容易被忽略的“内部状态”：KV cache 占用、队列深度、硬件异构。论文提出的 WVA autoscaler 不再像传统 HPA 那样只看 CPU 或副本数，而是把“模型现在卡在哪儿”也纳入决策。结果是有效吞吐提升 37%，请求失败降低 10 倍。给非技术读者打个比方：以前调度器像看停车场总车位，现在它开始看每辆车有没有熄火、出口堵不堵、哪条道更省油。LLM 服务之所以贵，很多时候不是算得慢，而是排队和缓存管理太笨。

05:00-06:00
工具使用与安全这块，`Governance Architecture for Autonomous Agent Systems`（arXiv:2603.07191 v2）是今天最像“工程规范”的一篇。它把 Agent 风险拆成四层：执行沙箱、意图校验、零信任 Agent 间授权、不可篡改审计日志。论文最有价值的地方，是它承认单一 guardrail 不够用：提示词注入、RAG 投毒、恶意插件并不是同一种病，不能指望一个分类器包治百病。实验里，多种本地 LLM judge 对恶意工具调用的拦截率达到 93% 到 98.5%，整套四层联动后，端到端 P50 延迟大约 980ms，其中非 judge 层只占约 18ms。意思很直接：安全不是“太贵所以先不做”，而是“设计得对，额外成本其实可控”。

06:00-07:00
把这几件事放在一起看，会出现一个很清晰的新趋势：AI 领域正在从“更强的模型”转向“更稳的系统”。Karpathy 在修自主研究循环的护栏；GLM-OCR 在为文档理解这类确定性任务做结构化提速；FlexServe 试图把安全隔离带进手机端推理；WVA 想办法让推理集群别把钱烧在错误的地方；LGA 则告诉大家，Agent 想进真实世界，必须先学会被约束。它们看上去分散，其实讲的是同一件事：下一轮竞争不只是模型榜单，而是谁能把模型、工具、设备和安全拧成一个能长期运行的系统。

07:00-08:00
最后说个判断：过去 24 小时里最值得普通开发者学习的，不是某个“更聪明”的新模型，而是这些项目共同体现出的工程意识。未来一年，多模态 Agent 会越来越像“会看文档、会调工具、会在边缘设备上运行的小型操作系统”；而真正拉开差距的，不是它偶尔多答对一道题，而是它能不能稳定跑一晚上、不泄露数据、不把 GPU 和手机电量白白耗光。研究正在变得越来越像造桥，不只是写诗。这反而是件好事：当 AI 开始认真面对摩擦、成本和安全，它才离日常生活更近。
