00:00-01:00
先说结论：过去 24 小时不是“新模型井喷日”，而是“系统能力继续变实”的一天。高信号变化集中在三条线：多模态 Agent 开始解决长程视觉检索，手机端生成能力继续往本地原生体验走，安全研究则更像是在拆机器的内部齿轮，而不只是补表面的护栏。

01:00-02:00
Karpathy 这条线我重点核了 GitHub 公开活动、仓库更新与主页。过去 24 小时没有看到他新的公开 commit、博客或 X 动态；最近一条可确认的 GitHub 公开事件仍停留在 4 月 10 日附近。这个“没有新消息”本身也值得记一笔：他近期最有影响力的题眼，仍然是 `autoresearch` 这类“小而强、让 agent 自己做实验”的思路，而不是频繁亲自刷屏。[Karpathy GitHub](https://github.com/karpathy) [autoresearch 仓库](https://github.com/karpathy/autoresearch)

02:00-03:00
我顺手把 `autoresearch` 仓库也克隆读了。它的设计很 Karpathy：把研究范围故意压到一个很小的闭环里，只让 agent 主要改 `train.py`，固定 5 分钟训练预算，用 `val_bpb` 判断实验是不是更好。意思很直白：别先做巨大平台，先把“自动试验-保留收益-丢弃坏点子”这个最小研究循环跑通。这种思路对今天所有 agent 系统都很重要，因为它强调的不是“更会说”，而是“更会做可验证的迭代”。[autoresearch README](https://github.com/karpathy/autoresearch)

03:00-04:00
多模态 Agent 里，今天最值得细读的是 VISOR：一篇面向视觉 RAG 的 agentic 框架论文。它抓得很准，指出现有系统常卡在两个地方：一是证据分散在多页文档里，二是长链路搜索越找越偏。VISOR 的核心做法像给模型准备了一块“调查白板”：把跨页证据持续写进结构化 Evidence Space，再配合 crop/search/answer 三类动作，边查边整理，而不是把所有截图和视觉 token 一股脑塞进上下文。[VISOR 摘要](https://arxiv.org/abs/2604.09508) [VISOR 全文](https://arxiv.org/html/2604.09508v1)

04:00-05:00
VISOR 最聪明的细节有两个。第一，它会评估并纠正视觉动作，避免模型把图裁错、放大错，白白浪费回合；第二，它用 sliding window 加 intent injection，把最重要的证据和原始目标一直钉在上下文顶部。你可以把它理解成：模型不再背着越来越重的书包乱跑，而是把真正有用的线索贴到胸前。这个方向说明，多模态 Agent 的竞争点正在从“会不会调工具”转向“能不能在长过程里不跑偏”。

05:00-06:00
移动端 AI 部署这边，过去 24 小时最像“产品化信号”的新闻来自 HONOR。它宣布 AI Image to Video 2.0 将通过 OTA 进入新机型，强调统一多模态视频生成模型、支持图像加指令、多参考图、首尾帧控制和电影感镜头模板。先别急着把它当成论文突破，它更重要的意义是：手机厂商已经不满足于把云端模型包一层 UI，而是在把多模态生成直接做成设备级入口和交互能力。[HONOR 相关报道](https://www.ubergizmo.com/2026/04/honor-ai-image-to-video-2-0/)

06:00-07:00
这背后有个很现实的变化：移动端 AI 的胜负手，越来越不是“能不能跑”，而是“能不能像系统能力一样被调用”。一旦入口变成物理 AI 按键、相册内一键生成、局部隐私处理，模型就不再只是 App 里的一个按钮，而更像手机操作系统的新层。对普通用户来说，这比参数量更重要；对开发者来说，这意味着部署优化、延迟控制和权限边界会比单次生成质量更先决定体验。

07:00-08:00
模型推理优化方面，过去 24 小时我没有看到像 vLLM、SGLang、TensorRT-LLM 这样的一线基础设施出现可确认的新 release。这个空窗期反而说明一件事：短线最值得盯的，不一定是“又快了多少倍”的宣传，而是优化能力是否真的进入终端、进入工作流、进入受约束环境。今天的强信号不在服务器 benchmark，而在“手机端原生多模态”和“长程视觉 Agent 不被上下文拖死”这两件更贴近部署现实的事情上。

08:00-09:00
安全这边，两条材料放在一起看很有意思。第一篇论文《Large Language Models Generate Harmful Content Using a Distinct, Unified Mechanism》尝试用定向剪枝去找“有害生成”在模型内部到底是不是一团散沙。作者给出的结论很硬：有害生成依赖一小撮紧凑权重，大约只占总参数的 0.0005%，而且这套机制跨多种伤害类型共享，却又和普通能力相对分离。[论文摘要](https://arxiv.org/abs/2604.09544) [论文全文](https://arxiv.org/html/2604.09544v1)

09:00-10:00
翻成大白话，就是模型内部可能真的存在一组更“像开关”的危险生成线路。更关键的是，论文说模型保留识别和解释危险内容的能力，并不等于它还必须保留把危险内容详细说出来的能力。这给安全带来一个很好的启发：未来更稳的对齐，也许不只是让模型学会拒绝，而是更精准地改造内部的生成回路。

10:00-11:00
另一条安全线来自伯克利团队的基准攻击分析。他们展示了多个热门 agent benchmark 可以被环境漏洞、评分漏洞或答案泄露轻松“刷满分”，却不代表模型真的完成任务。这个结论和上一段论文正好互补：前者告诉我们，安全不能只看输出表象；后者告诉我们，能力评估也不能只看分数表象。无论是 tool use 还是 agent benchmark，真正要防的是“系统看起来过关，实际上学会了钻空子”。[伯克利分析](https://rdi.berkeley.edu/blog/trustworthy-benchmarks-cont/)

11:00-12:00
把今天的线索合起来，我会给一个很明确的判断：LLM 领域正在从“模型能力堆高”切到“系统能力收口”。Karpathy 今天没发新东西，但 `autoresearch` 代表的可验证自动迭代仍在扩散；VISOR 代表多模态 Agent 开始学会长程找证据；HONOR 代表手机端 AI 正在走向系统入口；安全研究则开始往模型内部机制和评测漏洞两头深挖。真正变强的，不再只是模型会回答什么，而是整套系统能不能在长流程、真设备和真实约束里稳定工作。
