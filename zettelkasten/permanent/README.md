# Permanent 文件夹说明

这是**永久笔记 (Permanent Notes)** 的核心存储位置，也是 Zettelkasten 系统的**核心资产**。

## 永久笔记的定义

- **原子化**：每张笔记只包含**一个完整思想**
- **独立成篇**：不依赖其他笔记即可理解
- **去上下文化**：不引用特定来源（来源信息已转为链接）
- **可链接**：通过 `[[链接]]` 与相关笔记连接
- **长期保留**：永不删除，持续演化

## 命名格式

```
{ID}-{slugified-title}.md
# 示例：
001-zettelkasten-是什么.md
003-atomic-principle.md
```

**ID 规则**：
- 使用 3 位自增数字（001-999）
- 或时间戳（202602191200）
- 保持 ID 稳定，永不更改

**Title 规则**：
- 清晰、简洁、描述性强
- 小写、连字符分隔
- 可包含英文术语

## 关键原则

### 1. 原子性 ✅
每张笔记 = 一个完整思想单元

❌ 错误：
```
- AI发展历史
- Transformer原理
- GPT训练技巧
```
✅ 正确：拆成三张笔记

### 2. 链接至上 🔗
每张新笔记**必须**链接到至少一张已有笔记。

链接类型：
- `[[ID]]` 或 `[[ID-title]]`（Obsidian 自动补全）
- 不要只链接"相关"，要链接**具体相关点**

### 3. 自包含 💎
阅读此笔记不需要先读其他笔记（除非引用链接）。

如果感觉需要"前情提要"，说明笔记还不够原子化。

### 4. 去上下文化 🧹
不出现：
- "如前所述"
- "上述提到的X"
- "在这篇文章中"

转而：
- 直接陈述观点
- 用链接指向相关上下文（如果需要）

## 操作流程

### 创建新永久笔记

```bash
# 1. 在 inbox 或 literature 中发现有价值内容
# 2. 提炼原子观点 → 决定新笔记主题
# 3. 创建文件（手动或 obsidian-cli）
obsidian-cli create "permanent/016-new-note-title" --content "..."
# 4. 填写 front matter 和正文
# 5. 添加链接到相关笔记（至少1个）
# 6. 提交到 Git
```

### 链接最佳实践

✅ **好链接**：
```markdown
详见[[003-原子化原则]]，它强迫我们精确表达。
```
✅ **好链接（具体位置）**：
```markdown
关于[[006-自下而上-vs-自上而下]]中的涌现理论...
```
❌ **避免浮夸链接**：
```markdown
详见相关笔记 → 具体是哪个？
```

### 重构现有笔记

当笔记变得过长时：
- 拆分成多个原子笔记
- 创建主笔记聚合它们
- `obsidian-cli move` 会自动更新链接

### 搜索与发现

- **搜索**：`Cmd/Ctrl + P` 或 `obsidian-cli search-content "关键词"`
- **图谱**：打开图谱视图，发现主题簇
- **反向链接**：查看哪些笔记链接到当前笔记

## 质量检查清单

新永久笔记完成前：
- [ ] ID 唯一且稳定
- [ ] 标题清晰描述单一思想
- [ ] 内容原子化（不强行组合多个话题）
- [ ] 有至少 1 个 `[[链接]]` 指向已有笔记
- [ ] 有来源引用（如适用）
- [ ] 标签不过度（≤3个）
- [ ] 无 TODO 或占位符
