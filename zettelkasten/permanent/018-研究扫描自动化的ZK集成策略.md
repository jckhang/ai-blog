# 研究扫描自动化的 ZK 集成策略

将每日深度研究扫描（cron 任务）与 Zettelkasten 卡片系统结合，实现**自动知识捕获 → 原子化 → 长期积累**的闭环。

## 核心机制

**触发**: `0 0 * * *` (每日 00:00)
**输入**: 过去 24h AI 领域动态（通过 web_fetch, PDF 解析, git clone）
**输出**: 两种产物
1. 博客文章: `YYYY-MM-DD-llm-research-scan.md` (800-1500字, 科普风格)
2. 永久笔记: 3-5 张新卡片，直接写入 `permanent/`

## 自动化流程

```
research scan 运行
    ↓ 提取亮点
闪念笔记 (inbox/)  ← 同时写入
    ↓ Heartbeat review (次日)
永久笔记 (permanent/) ← 原子化提炼 (自动+人工)
    ↓ 链接化
[[链接]] 到相关卡片 (自动建议)
    ↓ 形成主题簇
博客文章（从卡片群串联）
```

## 关键技术点

### 输出格式

```json
{
  "date": "2026-02-19",
  "highlights": [
    {
      "id": "auto-20260219-001",
      "title": "Transformer 架构新突破",
      "content": "...",
      "tags": ["llm", "architecture"],
      "sources": ["https://arxiv.org/abs/..."],
      "related_cards": ["003", "016"]  // 自动计算相似度
    }
  ]
}
```

### 自动链接建议

- 使用 embedding 计算新亮点与已有卡片的语义相似度
- 取 top-3 相似卡片建议链接
- 最终链接需人工确认（避免过度链接）

### 标签策略

- 固定标签: `research`, `dd:YYYY-MM-DD`
- 动态标签: 基于内容的主题（`llm`, `agent`, `multimodal`）
- 来源标签: `source:arxiv`, `source:github`, `source:blog`
